บทที่ 2

FOR Loop

✅ RF 4.0+ (IN ENUMERATE start=: RF 6.0+)

🎯 อ่านจบแล้วทำได้

  • วน loop ผ่าน list ด้วย FOR IN
  • วนตามตัวเลขด้วย FOR IN RANGE
  • ดึง index พร้อมค่าด้วย FOR IN ENUMERATE
  • วน 2 list คู่ขนานด้วย FOR IN ZIP

FOR IN — วน list ทีละ item

ใช้เมื่อมี list อยู่แล้วและต้องการทำอะไรกับทุก item

FOR IN — วนผ่าน list ผลไม้

FOR IN RANGE — วนตามตัวเลข

Syntaxได้ค่า
IN RANGE 50, 1, 2, 3, 4
IN RANGE 1 61, 2, 3, 4, 5
IN RANGE 0 10 20, 2, 4, 6, 8
IN RANGE 10 0 -110, 9, 8 ... 1

ค่า end ไม่ถูกรวม — เหมือน Python range() ทุกอย่าง

FOR IN RANGE — 0 ถึง 4

FOR IN ENUMERATE — วนพร้อม index

เมื่อต้องรู้ว่า item อยู่ลำดับที่เท่าไหร่ เช่น log "แถวที่ N" หรือ validate ตาราง

@{items}=    Create List    alpha    beta    gamma

# index เริ่มที่ 0 (default)
FOR    ${index}    ${item}    IN ENUMERATE    @{items}
    Log    [${index}] ${item}
END
Output
[0] alpha
[1] beta
[2] gamma
# เริ่ม index จาก 1 (RF 6.0+)
FOR    ${index}    ${item}    IN ENUMERATE    @{items}    start=1
    Log    [${index}] ${item}
END
Output
[1] alpha
[2] beta
[3] gamma

FOR IN ZIP — วน 2 list พร้อมกัน

ใช้จับคู่ข้อมูลจาก 2 list เช่น ชื่อ + คะแนน, input + expected output

${names}=    Create List    Alice    Bob    Carol
${scores}=   Create List    90       75     88
FOR    ${name}    ${score}    IN ZIP    ${names}    ${scores}
    Log    ${name}: ${score}
END
Output
Alice: 90
Bob: 75
Carol: 88
ข้อจำกัด IN ZIP: ต้องใช้ตัวแปร list เท่านั้น ไม่รับ inline string — IN ZIP Alice Bob 1 2 จะ error
และถ้า list ยาวไม่เท่ากัน จะวนตาม list ที่สั้นกว่า items ที่เหลือจะถูกข้ามโดยไม่แจ้งเตือน

เลือก variant ไหนดี?

สถานการณ์ใช้
วน list ไม่ต้องรู้ indexFOR IN
วนตามจำนวนรอบที่รู้ล่วงหน้าFOR IN RANGE
ต้องรู้ว่า item อยู่ตำแหน่งที่เท่าไหร่FOR IN ENUMERATE
จับคู่ข้อมูลจาก 2 listFOR IN ZIP

Common Mistakes

❌ ใช้ ${} แทน @{} — ได้ list ทั้งก้อนเป็น item เดียว
FOR    ${x}    IN    ${myList}    # วนแค่รอบเดียว!
✅ ใช้ @{} เพื่อ unpack
FOR    ${x}    IN    @{myList}
source: RF User Guide — "@ prefix unpacks the list"
❌ IN RANGE ใส่ expression ตรงๆ
FOR    ${i}    IN RANGE    ${count} + 1    # ไม่ทำงาน
✅ Evaluate ก่อน
${end}=    Evaluate    ${count} + 1
FOR    ${i}    IN RANGE    ${end}
source: RF User Guide — IN RANGE รับแค่ตัวเลขหรือ variable เดี่ยว

🧠 เช็คความเข้าใจ

ข้อ 1. FOR ${i} IN RANGE 2 8 3 — ${i} จะมีค่าอะไรบ้าง?

ดูเฉลย
2, 5 — เริ่มที่ 2, บวก step 3 ทุกรอบ, หยุดก่อนถึง 8

ข้อ 2. FOR IN ZIP หยุดเมื่อไหร่ถ้า list A มี 4 items, list B มี 2 items?

ดูเฉลย
วน 2 รอบ แล้วหยุด เพราะ ZIP หยุดเมื่อ list ที่สั้นกว่าหมด — items ที่เหลือใน A ถูกข้ามไปเงียบๆ
