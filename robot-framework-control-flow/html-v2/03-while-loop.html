บทที่ 3

WHILE Loop, BREAK และ CONTINUE

⚠️ RF 5.0+

🎯 อ่านจบแล้วทำได้

  • เขียน WHILE loop ที่วนจนเงื่อนไขเปลี่ยน
  • ตั้ง limit ป้องกัน infinite loop
  • ใช้ BREAK ออกจาก loop กลางคัน
  • ใช้ CONTINUE ข้าม iteration ที่ไม่ต้องการ

WHILE ต่างจาก FOR ยังไง?

FOR ใช้เมื่อรู้จำนวนรอบล่วงหน้า — WHILE ใช้เมื่อไม่รู้ เช่น รอ API response, retry จนสำเร็จ

WHILE เช็ค condition ก่อน ทุกรอบ — ถ้า False ตั้งแต่แรก loop body ไม่ทำงานเลย

ดูการทำงาน — counter < 3

WHILE loop — counter < 3

Limit — ป้องกัน Infinite Loop

WHILE มี limit default = 10,000 รอบ เมื่อถึงจะ FAIL ทันที ปรับได้ด้วย parameter limit:

รูปแบบความหมาย
WHILE cond limit=100สูงสุด 100 รอบ
WHILE cond limit=30 secondsสูงสุด 30 วินาที
WHILE cond limit=NONEไม่มี limit — ต้องมั่นใจว่ามี BREAK

BREAK — ออกจาก Loop ทันที

BREAK — หยุดเมื่อเจอเลข 4

CONTINUE — ข้าม Iteration นี้

CONTINUE — ข้ามเลขคู่

Common Mistakes

❌ ลืมอัปเดต condition variable → infinite loop
${x}=    Set Variable    0
WHILE    ${x} < 10
    Log    วนตลอดกาล
    # ลืม update ${x} → วนจนถึง 10,000 แล้ว FAIL
END
✅ อัปเดตตัวแปรใน loop body เสมอ
    ${x}=    Evaluate    ${x} + 1
source: RF User Guide
❌ BREAK / CONTINUE นอก loop
BREAK    # error — ต้องอยู่ใน FOR หรือ WHILE เท่านั้น
✅ ใช้ได้เฉพาะภายใน loop body
source: RF User Guide

🧠 เช็คความเข้าใจ

ข้อ 1. ถ้า WHILE condition เป็น False ตั้งแต่แรก loop body ทำงานกี่รอบ?

ดูเฉลย
0 รอบ — RF เช็ค condition ก่อนทุกรอบ ถ้า False ตั้งแต่แรกข้ามทั้ง loop ไปเลย

ข้อ 2. ต่างกันยังไงระหว่าง BREAK กับ CONTINUE?

ดูเฉลย
BREAK หยุด loop ทั้งหมด ออกไปรัน keyword ถัดจาก END | CONTINUE หยุดแค่ iteration ปัจจุบัน แล้วไปเริ่มรอบถัดไป

ข้อ 3. เมื่อไหร่ควรใช้ WHILE แทน FOR?

ดูเฉลย
เมื่อไม่รู้จำนวนรอบล่วงหน้า เช่น รอ element โหลด, poll API จน status เปลี่ยน, retry จนสำเร็จ
