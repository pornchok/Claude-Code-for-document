Glossary คำศัพท์
AI & Machine Learning สำหรับผู้เริ่มต้น

📚 34 คำศัพท์ 🔍 ค้นหาได้ 🗂️ กรองตามบท

แสดง 34 คำศัพท์

AI (Artificial Intelligence)
บทที่ 1
แนวคิดกว้างที่หมายถึงระบบคอมพิวเตอร์ที่สามารถทำงานที่ปกติต้องใช้สติปัญญาของมนุษย์ เช่น จดจำภาพ เข้าใจภาษา หรือตัดสินใจ AI ไม่ได้หมายความว่าเครื่องจักร "คิดได้" แต่หมายถึงทำงานบางอย่างได้โดยอัตโนมัติจากการเรียนรู้ pattern — AI เป็น umbrella term ที่ครอบคลุม Machine Learning, Deep Learning, และ LLM
Attention Mechanism
บทที่ 2
กลไกสำคัญใน Transformer ที่ช่วยให้ model "ให้ความสำคัญ" กับส่วนต่างๆ ของ input ต่างกัน เมื่อแปลประโยค model จะ "attend" ไปยังคำที่เกี่ยวข้องมากที่สุดในการแปลแต่ละคำ เปรียบเสมือนการที่คนอ่านย้อนไปหาส่วนที่เกี่ยวข้องขณะเขียนสรุป — กระดาษ "Attention Is All You Need" (2017) เป็นจุดเริ่มต้นของ Transformer
Autoregressive
บทที่ 2
วิธีการ generate text ของ LLM ที่สร้าง output ทีละ token โดยใช้ tokens ก่อนหน้าทั้งหมดเป็น input สำหรับ token ถัดไป เหมือนกับการเดาคำถัดไปในประโยคโดยอ่านทุกอย่างที่เขียนมาแล้ว ผลคือ LLM ไม่สามารถ "แก้ไข" คำก่อนหน้าได้หลังจาก generate ออกไปแล้ว
Bias
บทที่ 5
ความลำเอียงที่ฝังอยู่ใน AI model เนื่องจากเรียนรู้จาก training data ที่ reflect อคติของสังคมมนุษย์ ตัวอย่างเช่น model อาจมี gender bias ในการแนะนำอาชีพ หรือ perform แย่กว่าสำหรับภาษาที่มีข้อมูลน้อย Bias เป็น systematic error ที่มี pattern สม่ำเสมอ ต่างจาก hallucination ที่เป็น random error
Chain-of-Thought (CoT)
บทที่ 3
เทคนิค prompting ที่บอกให้ AI "คิดทีละขั้นตอน" ก่อนตอบ แทนที่จะกระโดดไปยังคำตอบทันที วิธีนี้ช่วยให้ AI handle งานที่ต้องการ reasoning ซับซ้อน เช่น math problems หรือ logical analysis ได้ดีกว่ามาก ตัวอย่าง prompt: "คิดทีละขั้นตอนก่อนตอบ" หรือ "Let's think step by step"
Claude
บทที่ 4
AI assistant ที่พัฒนาโดย Anthropic ออกแบบมาให้เป็น helpful, harmless, and honest (3H) Claude ใช้ Constitutional AI ในการ training เพื่อให้ค่านิยมของ AI align กับค่านิยมของมนุษย์ มีระบบ Skills (Superpowers) ที่ช่วยให้ปรับ behavior ได้ตาม workflow ที่ต้องการ
Confabulation (Hallucination)
บทที่ 5
คำทางการสำหรับ hallucination ใน AI — การที่ model สร้างข้อมูลที่ "ฟังดูน่าเชื่อ" แต่ไม่มีอยู่จริง คำว่า confabulation มาจากจิตวิทยา หมายถึงการที่สมองสร้าง memory ปลอมโดยไม่รู้ตัว ใน AI เกิดจาก generation process ที่ optimize สำหรับ plausibility ไม่ใช่ accuracy
Context Window
บทที่ 2
ขนาดสูงสุดของ tokens ที่ LLM สามารถ "มองเห็น" และ process ได้พร้อมกันในครั้งเดียว เปรียบเหมือน "ความจำระยะสั้น" ของ AI ถ้า conversation หรือ document ยาวเกินขีดจำกัด ข้อมูลตอนต้นจะถูกลืม Claude รุ่นล่าสุดมี context window ขนาดใหญ่มาก (หลาย hundred thousand tokens)
Deep Learning
บทที่ 1
ส่วนหนึ่งของ Machine Learning ที่ใช้ Neural Network หลายชั้น (deep = หลายชั้น) เรียนรู้ pattern ซับซ้อนโดยอัตโนมัติ เก่งมากกับ image recognition, speech recognition, และ natural language processing ต้องการ training data และ computing power มากกว่า ML แบบดั้งเดิม
Embedding
บทที่ 2
การแปลง tokens (คำหรือข้อความ) ให้เป็น vector ของตัวเลข เพื่อให้ AI สามารถประมวลผล "ความหมาย" ได้ คำที่มีความหมายใกล้เคียงกันจะมี embedding vectors ที่อยู่ใกล้กันใน vector space เช่น "กษัตริย์" และ "พระราชา" จะมี embedding ที่ใกล้กัน ส่วน "กษัตริย์" และ "ปลา" จะอยู่ห่างกัน
Few-shot Learning
บทที่ 3
เทคนิค prompting ที่ให้ตัวอย่าง input-output จำนวนน้อย (2-5 ตัวอย่าง) ใน prompt เพื่อสอน AI ว่าต้องการ output แบบไหน ช่วยให้ AI เข้าใจ format, tone, และ style ที่ต้องการโดยไม่ต้องอธิบายยาว เหมาะกับงาน classification และ transformation ที่มี clear pattern
Fine-tuning
บทที่ 2
กระบวนการ train model เพิ่มเติมด้วย dataset เฉพาะทาง หลังจาก pre-training บน general data แล้ว เช่น เอา foundation model มา fine-tune ด้วยข้อมูลทางการแพทย์เพื่อให้ตอบคำถามทางการแพทย์ได้ดีขึ้น Fine-tuning ใช้ทรัพยากรน้อยกว่า training จาก scratch มาก
Foundation Model
บทที่ 2
model ขนาดใหญ่ที่ train ด้วยข้อมูลหลากหลายจำนวนมหาศาล จนสามารถทำงานได้หลายอย่าง ซึ่งใช้เป็น "ฐาน" ในการพัฒนาต่อสำหรับ use cases เฉพาะทาง เช่น Claude, GPT-4, Gemini ล้วนเป็น foundation models ที่สามารถนำไป fine-tune หรือ customize สำหรับงานเฉพาะได้
Generative AI
บทที่ 1
ประเภทของ AI ที่สร้าง content ใหม่ได้ ไม่ว่าจะเป็น text, image, audio, video, หรือ code ต่างจาก AI แบบเดิมที่แค่ classify หรือ predict จากข้อมูลที่มี Generative AI สามารถสร้างสิ่งที่ไม่เคยมีมาก่อนได้ ตัวอย่าง: Claude (text), DALL-E (image), Suno (music)
Hallucination
บทที่ 5
พฤติกรรมที่ AI สร้างข้อมูลที่ฟังดูน่าเชื่อถือแต่ไม่ถูกต้องหรือไม่มีอยู่จริง เช่น อ้าง citation ที่ไม่มี บอกตัวเลขผิด หรืออธิบายกฎหมายที่ไม่มีอยู่ Hallucination เกิดจาก generation process ที่ optimize สำหรับ plausibility ไม่ใช่ accuracy ทำให้ AI ตอบด้วยความมั่นใจแม้ข้อมูลผิด
Inference
บทที่ 2
กระบวนการที่ trained model สร้าง output จาก input ที่ได้รับ ตรงข้ามกับ training ซึ่งเป็นการปรับ weights ของ model เมื่อคุณพิมพ์คำถามให้ Claude และรอคำตอบ นั่นคือ inference — model ใช้ weights ที่ train มาแล้วในการสร้างคำตอบ Inference ใช้ทรัพยากรน้อยกว่า training มาก
LLM (Large Language Model)
บทที่ 1–2
ส่วนหนึ่งของ Deep Learning ที่เน้นการเข้าใจและสร้างภาษาธรรมชาติ train ด้วยข้อมูลข้อความขนาดใหญ่มหาศาล ใช้ Transformer architecture และ Attention Mechanism ตัวอย่าง: Claude (Anthropic), GPT-4 (OpenAI), Gemini (Google), Llama (Meta) — นี่คือเทคโนโลยีที่อยู่เบื้องหลัง AI chatbot ที่เราใช้อยู่ทุกวันนี้
Machine Learning (ML)
บทที่ 1
ส่วนหนึ่งของ AI ที่ให้คอมพิวเตอร์เรียนรู้จากข้อมูลโดยไม่ต้อง hard-code ทุก rule ด้วยมือ แทนที่จะเขียนว่า "ถ้าเมลมีคำว่า FREE แล้วขึ้นต้นด้วย $$ ให้เป็น spam" ML เรียนรู้เองจากตัวอย่าง spam นับล้านอีเมล ครอบคลุม: supervised learning, unsupervised learning, reinforcement learning
Neural Network
บทที่ 1–2
โครงสร้างพื้นฐานของ Deep Learning ที่ได้แรงบันดาลใจมาจากโครงสร้างสมองมนุษย์ ประกอบด้วย "neurons" (nodes) เชื่อมต่อกันเป็นชั้นๆ (layers) แต่ละ connection มี weight ที่ปรับได้ระหว่าง training เมื่อ network ลึก (หลาย layers) เรียกว่า Deep Neural Network ซึ่งเป็นรากฐานของ Deep Learning
Parameters
บทที่ 2
ตัวเลขทั้งหมดใน model ที่ถูกเรียนรู้ระหว่าง training ใน Neural Network คือ weights ของ connections ระหว่าง neurons ยิ่ง parameters มาก model มักจะ powerful กว่าแต่ต้องการ data และ compute มากขึ้น GPT-3 มี ~175 billion parameters, Claude มีมากกว่านั้น ขนาด parameters มักใช้เป็น indicator ของ model capability
Priority Order (Claude Superpowers)
บทที่ 4
ลำดับชั้นที่กำหนดว่า instruction ใดมีอำนาจเหนือกว่าเมื่อขัดแย้งกัน: User (สูงสุด) → Skill → Default System Prompt (ต่ำสุด) ออกแบบมาเพื่อให้ user มีอำนาจควบคุมสูงสุดเสมอ ขณะที่ Skill ช่วยกำหนด standard สำหรับ workflow และ Default ทำหน้าที่เป็น safety net
Prompt
บทที่ 3
ข้อความ input ที่คุณส่งให้ AI เป็น "คำสั่ง" หรือ "คำถาม" ที่กำหนดว่า AI จะตอบอะไร Prompt ที่ดีจะให้ Role, Context, Task, Format, และ Constraint ที่ชัดเจน คุณภาพของ Prompt มีผลโดยตรงต่อคุณภาพของ output ที่ได้จาก AI
Prompt Engineering
บทที่ 3
ทักษะและศาสตร์ในการออกแบบ prompt ที่ได้ผลลัพธ์ดีที่สุดจาก AI ครอบคลุมการกำหนด role, context, task, format, constraint, การใช้ few-shot examples, chain-of-thought, และ iterative refinement ไม่ใช่การ "หลอก" AI แต่เป็นการสื่อสารที่ชัดเจนและมีประสิทธิภาพ
RLHF (Reinforcement Learning from Human Feedback)
บทที่ 2
เทคนิคที่ใช้ feedback จากมนุษย์ในการปรับให้ AI ตอบสนองตาม human preference มากขึ้น มนุษย์ rate ว่า response ไหนดีกว่า AI เรียนรู้จาก preferences เหล่านั้น เป็นส่วนสำคัญที่ทำให้ Claude และ ChatGPT "เป็นมิตร" และ "มีประโยชน์" มากกว่า base model ที่ train แค่จากข้อมูล internet ล้วนๆ
Skill (Claude Superpowers)
บทที่ 4
ชุดของ instructions และ configurations ที่กำหนดพฤติกรรมของ Claude สำหรับ workflow เฉพาะ วางไว้ระหว่าง User และ Default system prompt ในลำดับ priority Skill ช่วยให้ทีมสามารถ standardize วิธีใช้ AI โดยไม่ต้องเขียน prompt ยาวๆ ซ้ำทุกครั้ง และ update ได้ที่จุดเดียว
System Prompt
บทที่ 4
ชุดของ instructions ที่ถูกส่งให้ AI ก่อนการสนทนาจะเริ่ม โดยปกติ user จะไม่เห็น ใช้กำหนด persona, rules, และ context ของ AI ใน application นั้นๆ เช่น chatbot บริษัทอาจมี system prompt ที่บอกว่า "คุณคือ assistant ของบริษัท X ตอบเฉพาะเรื่องผลิตภัณฑ์ของเรา" — System Prompt อยู่ใน priority ระดับต่ำสุดใน Claude Superpowers
Temperature
บทที่ 2
parameter ที่ควบคุมความ "สุ่ม" ของการ generate — Temperature สูง (เช่น 0.9) ทำให้ output สร้างสรรค์และหลากหลายกว่า แต่อาจ random มากขึ้น Temperature ต่ำ (เช่น 0.1) ทำให้ output สม่ำเสมอและ predictable กว่า เหมาะกับงานที่ต้องการความแม่นยำสูง เช่น code generation หรือการตอบคำถามข้อเท็จจริง
Token
บทที่ 2
หน่วยพื้นฐานที่ LLM ใช้ประมวลผลข้อความ ไม่ใช่ตัวอักษร ไม่ใช่คำ แต่เป็นชิ้นส่วนของข้อความที่ขนาดแตกต่างกัน โดยประมาณ: ภาษาอังกฤษ 1 token ≈ 4 ตัวอักษร หรือ 1 คำสั้น, ภาษาไทย 1 คำ ≈ 2-3 tokens ขึ้นกับ tokenizer (ข้อความไทยยาวเท่ากันมักใช้ tokens มากกว่าภาษาอังกฤษ 2-3 เท่า) — Claude ทั้ง input และ output นับเป็น tokens ซึ่งส่งผลต่อ context window และค่าใช้จ่าย
Tokenization
บทที่ 2
กระบวนการแบ่งข้อความออกเป็น tokens ก่อนส่งให้ model ประมวลผล LLM ใหม่ๆ ส่วนใหญ่ใช้ Byte Pair Encoding (BPE) ซึ่งสร้าง vocabulary ของชิ้นส่วนคำที่พบบ่อย ภาษาที่มีข้อมูลมากใน training จะได้ vocabulary ที่ efficient กว่า ทำให้ใช้ tokens น้อยกว่าต่อความหมายเท่ากัน
Training Data
บทที่ 2
ชุดข้อมูลที่ใช้ "สอน" AI ในกระบวนการ training คุณภาพและความหลากหลายของ training data มีผลโดยตรงต่อ capability และ bias ของ model LLM ขนาดใหญ่ train ด้วยข้อมูลจาก internet, หนังสือ, งานวิจัย จำนวนนับพัน billion tokens training data ที่มี bias จะทำให้ model มี bias ตามไปด้วย
Transformer
บทที่ 2
architecture ของ Neural Network ที่ใช้ Attention Mechanism เป็นหัวใจหลัก แทนที่จะอ่านข้อความทีละตัวอักษรแบบลำดับ Transformer มองทุกส่วนของ input พร้อมกันและประเมินว่าแต่ละส่วน "เกี่ยวข้อง" กันอย่างไร เกิดจากกระดาษ "Attention Is All You Need" (Google, 2017) และเป็นรากฐานของ LLM ทุกตัวในปัจจุบัน
Vector
บทที่ 2
รายการของตัวเลขที่ใช้แทน "ความหมาย" หรือ "คุณสมบัติ" ของข้อมูลใน AI Embedding แปลง tokens เป็น vectors เพื่อให้ AI ประมวลผลได้ ตัวอย่างเช่น คำว่า "แมว" อาจถูกแทนด้วย vector [0.2, -0.8, 0.5, ...] ซึ่งมีคุณสมบัติที่ใกล้เคียงกับ "สัตว์เลี้ยง" มากกว่า "ยานพาหนะ"
Weights
บทที่ 2
ตัวเลขในแต่ละ connection ของ Neural Network ที่ถูกปรับค่าระหว่าง training กระบวนการ training คือการปรับ weights ทีละน้อยเพื่อให้ prediction ของ model ถูกต้องมากขึ้น เมื่อ training เสร็จ weights จะถูก "freeze" และนำไปใช้ใน inference Weights คือ "ความรู้" ที่ AI เรียนมา
Zero-shot Learning
บทที่ 3
ความสามารถของ LLM ในการทำงานตาม instruction โดยไม่ต้องให้ตัวอย่างเลย เช่น "แปลข้อความนี้เป็นภาษาญี่ปุ่น" โดยไม่ต้องบอกว่าการแปลที่ดีหน้าตาเป็นอย่างไร เป็นความสามารถที่เกิดจาก training ด้วยข้อมูลขนาดใหญ่ ต่างจาก Few-shot ที่ต้องให้ตัวอย่างก่อน
