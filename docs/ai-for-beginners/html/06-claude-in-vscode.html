🎯 อ่านจบบทนี้แล้ว คุณจะทำได้

🤔 ทำไมต้องรู้เรื่องนี้?

💡 Why ก่อน What

หลายคนรู้จัก Claude จาก claude.ai — เปิด browser แล้วถามได้เลย ง่ายมาก แต่ถ้าคุณเป็นนักพัฒนาที่ต้องการให้ Claude เห็น codebase จริงของคุณ, แก้ไขไฟล์, รัน terminal และใช้ Skills เต็มรูปแบบ — claude.ai web ทำไม่ได้

บทนี้จะไขข้อสงสัยว่า Claude มีให้ใช้ตรงไหนบ้าง แต่ละที่ทำอะไรได้ต่างกันอย่างไร และเส้นทางที่เร็วที่สุดสำหรับนักพัฒนาคือ VS Code extension ซึ่งให้พลังเต็มรูปแบบของ Claude Code โดยไม่ต้องทิ้ง editor ที่คุณใช้อยู่แล้ว

💡 Analogy: Claude เหมือน Photoshop

🎭 อุปมา

ลองนึกถึง Adobe Photoshop — คุณสามารถแก้รูปบน Photoshop.com (web), ติดตั้ง Photoshop desktop app (เต็มรูปแบบ) หรือใช้ extension ใน Lightroom ที่คุณทำงานอยู่แล้ว

Claude ก็เหมือนกัน: Claude.ai web = Photoshop.com ที่เปิดได้ทันทีแต่ feature น้อยกว่า, Claude Code CLI = desktop app เต็มรูปแบบ, VS Code extension = plugin ที่เอา Photoshop มาฝังใน workflow ที่คุณทำงานอยู่แล้ว

⚠️ ถ้าเชื่อ analogy นี้ 100% จะเข้าใจผิดว่า...
  • VS Code extension เป็นแค่ "plugin" ที่ feature น้อยกว่า CLI — ในความเป็นจริง extension รัน Claude Code ที่ backend เดียวกับ CLI ทุกประการ Skills ทำงานครบเหมือนกัน 100% ไม่มีการตัด feature ออก
  • Web เป็นแค่เวอร์ชันเล็กของ app เดียวกัน — จริงๆ claude.ai web เป็น product คนละตัวที่ออกแบบสำหรับ general chat ไม่ใช่ Claude Code ที่ออกแบบสำหรับนักพัฒนาโดยเฉพาะ ความสามารถต่างกันในระดับสถาปัตยกรรม
  • ต้อง install ครบทุก platform — ไม่จริง เลือกได้ตามงาน VS Code extension เพียงตัวเดียวก็ครบสำหรับ developer workflow ส่วนใหญ่

💻 VS Code Extension — สำหรับ Developer ทุกระดับ ⭐

⭐
นี่คือ highlight ของบทนี้ — VS Code extension ให้พลังของ Claude Code ทั้งหมด แต่ integrate เข้ากับ editor ที่คุณทำงานอยู่แล้ว เหมาะสำหรับนักพัฒนาทุกระดับโดยไม่ต้องเปลี่ยน workflow
💻
Claude Code — VS Code Extension
Anthropic Claude · Visual Studio Marketplace

วิธีติดตั้ง (ทีละขั้น)

  1. เปิด VS Code

    ต้องเป็น VS Code version 1.80+ ขึ้นไป ถ้ายังไม่ได้ update แนะนำ update ก่อน

  2. เปิด Extensions panel

    กด Ctrl+Shift+X (Windows/Linux) หรือ Cmd+Shift+X (Mac)

  3. ค้นหา "Claude Code"

    พิมพ์ "Claude Code" หรือ "Anthropic Claude" ในช่องค้นหา เลือก extension โดย Anthropic

  4. กด Install

    หรือเปิดผ่าน Marketplace URL ได้เลย: https://marketplace.visualstudio.com/items?itemName=anthropic.claude-code

  5. Sign in ด้วย Anthropic account

    VS Code จะถามให้ authorize — กด Allow แล้ว login ผ่าน browser ที่เปิดขึ้นมา

วิธีใช้งานหลัก

1. เปิด Chat Panel

กด Ctrl+Shift+P → พิมพ์ "Claude Code" หรือกดที่ icon Claude ใน Activity Bar (แถบซ้ายสุดของ VS Code)

2. Chat กับ Claude ได้เลย

Claude เห็นไฟล์ที่คุณเปิดอยู่และ context ทั้งโปรเจค เลือก code แล้วถาม "explain this" หรือ "fix this bug" ได้เลยโดยไม่ต้อง copy-paste อะไร

3. Inline Code Actions

เลือก code block ที่ต้องการ → กด Ctrl+Shift+A → เลือก action ที่ต้องการ: Explain / Fix / Refactor / Test

4. Invoke Skills / Superpowers

พิมพ์ /ชื่อ-skill ใน chat panel — ทำงานเหมือน CLI เลย 100%

/systematic-debugging /code-review /brainstorming /test-driven-development

🎓 ตัวอย่าง 3 ระดับ

Beginner — เริ่มต้นครั้งแรก

นักศึกษาลองใช้ Claude ช่วยเขียน Python script ครั้งแรก

แพนนาเรียน Python เพิ่งเริ่มต้น อยากลองให้ Claude ช่วยเขียน script ง่ายๆ เธอเคยใช้แต่ claude.ai web ซึ่ง copy-paste code ไปมาน่าเบื่อมาก

เธอลองติดตั้ง VS Code extension ครั้งแรก:

  1. กด Ctrl+Shift+X ค้นหา "Claude Code" แล้ว install
  2. เปิดโปรเจคของเธอ สร้างไฟล์ hello.py
  3. เปิด Claude chat panel แล้วพิมพ์ว่า "ช่วยเขียน script คำนวณค่าเฉลี่ยจาก list ของตัวเลขหน่อย"

Claude ตอบกลับมาทันทีพร้อม code สมบูรณ์ เธอกด "Insert at cursor" — code ถูก insert เข้าไฟล์โดยตรง ไม่ต้อง copy-paste เลย

💡
สิ่งที่เปลี่ยน: จาก "ต้อง copy-paste ไปมาระหว่าง browser กับ editor" → เป็น "Claude อยู่ใน editor เดียวกันกับ code ของเราเลย" ดูง่ายกว่ามาก
Intermediate — สถานการณ์จริง

Junior Developer debug ปัญหา API authentication ด้วย /systematic-debugging

ต่อมีโปรเจค REST API ที่ return 401 Unauthorized แบบไม่สม่ำเสมอ — บางครั้งใช้ได้ บางครั้งไม่ได้ เขา reproduce ไม่ได้เลย เสียเวลา debug ไปแล้ว 2 ชั่วโมง

เขาเปิด Claude chat ใน VS Code แล้วพิมพ์:

/systematic-debugging API return 401 แบบ intermittent ไม่ consistent reproduce ไม่ได้เลย

Claude (ผ่าน /systematic-debugging Skill) ถามคำถามเป็นระบบ:

  1. "Token expire หลังกี่นาที? มี refresh mechanism ไหม?"
  2. "เกิดตอน load สูงไหม หรือสุ่มๆ?"
  3. "เป็น race condition ระหว่าง request ไหม? ลองดู timestamp ของ request ที่ fail"

จากคำถามข้อ 3 ต่อตรวจสอบแล้วพบว่ามี race condition — request 2 ตัวส่งพร้อมกันและ token ถูก invalidate ไปก่อน Claude แนะนำให้ implement token locking และ retry logic ซึ่ง fix ปัญหาได้ทันที

💡
Key point: /systematic-debugging ไม่ได้แค่ "ตอบคำถาม" แต่นำ process debug แบบ expert มาให้ ซึ่ง Claude ธรรมดาไม่ได้ทำแบบนี้โดยอัตโนมัติ
Advanced — Production Grade

Senior Dev ใช้ Claude Code เป็นส่วนหนึ่งของ pre-commit workflow

ทีม 5 คนทำงานร่วมกันใน monorepo ขนาดใหญ่ code quality ไม่สม่ำเสมอเพราะ review ใช้เวลานานและบางครั้งข้ามผ่าน minor issues

สิ่งที่ทีมทำ: integrate /code-review Skill เข้า workflow ก่อน commit ทุกครั้ง

ทุกคนในทีม configure VS Code extension + CLAUDE.md ในโปรเจค ซึ่งกำหนด standard ของทีมไว้ล่วงหน้า เช่น:

# Team Standards (CLAUDE.md)
- ทุก function ต้องมี error handling
- ห้ามใช้ console.log ใน production code
- ทุก async function ต้องมี try/catch

ก่อน commit ทุกครั้ง developer เปิด Claude chat แล้วพิมพ์:

/code-review ช่วย review ไฟล์นี้ก่อน commit

Claude อ่าน CLAUDE.md และ review ตาม standard ของทีม report ออกมาพร้อม severity: Critical / Warning / Info developer แก้ Critical และ Warning ก่อน commit เสมอ

🔬
ผลลัพธ์: ภายใน 1 เดือน ทีมพบว่า bugs ที่หลุดไป production ลดลง 60% เพราะ issues ส่วนใหญ่ถูกจับได้ตั้งแต่ก่อน commit — CLAUDE.md คือสิ่งที่ทำให้ทีมได้ "AI ที่รู้ standards ของทีมเรา" ไม่ใช่แค่ AI ทั่วไป

❓ คำถามที่ถามบ่อย

ถ้าใช้ VS Code extension แล้ว Skills ใช้ได้เหมือนกันไหม?
✅ ใช่ — Skills ทำงานเหมือนกัน 100% VS Code extension รัน Claude Code ที่ backend เดียวกันกับ CLI ทุกประการ Skills ทุกตัวไม่ว่าจะเป็น /systematic-debugging, /code-review, /brainstorming หรืออื่นๆ ทำงานครบเหมือนกัน ไม่มีการตัด feature ออก
ต้อง install Claude Code CLI ก่อนไหมถึงจะใช้ VS Code extension ได้?
ไม่จำเป็นสำหรับการใช้งานพื้นฐาน VS Code extension ติดตั้งได้โดยตรงและทำงานได้เองเลย อย่างไรก็ตาม ถ้าต้องการ customize CLAUDE.md หรือใช้งาน automated scripts จาก terminal ก็ควร install CLI เพิ่มด้วย
Claude Code VS Code extension ฟรีไหม?
Extension ตัวเองฟรี — ดาวน์โหลดและ install ได้ฟรี แต่การใช้งานต้องใช้ Anthropic API key หรือ Claude Code subscription ซึ่งมีค่าใช้จ่ายตามการใช้งาน ดู pricing ล่าสุดได้ที่ anthropic.com/pricing
ใช้ได้กับ VS Code fork อื่นๆ เช่น Cursor หรือ Windsurf ไหม?
Cursor มี built-in AI ของตัวเองที่ integrate กับ Claude ในระดับหนึ่ง แต่ไม่ใช่ Claude Code extension โดยตรง — Skills System ไม่ทำงานใน Cursor เหมือนกับใน VS Code native

Claude Code extension ออกแบบสำหรับ VS Code (Microsoft) โดยเฉพาะ บาง fork ที่ compatible กับ VS Code extension อาจ install ได้แต่อาจมี limitation บางส่วน
JetBrains ได้ไหม? (IntelliJ, PyCharm, WebStorm)
ได้ — มี Claude Code plugin สำหรับ JetBrains ติดตั้งผ่าน JetBrains Marketplace โดยไปที่ Settings (Ctrl+Alt+S) → Plugins → Marketplace แล้วค้นหา "Claude Code" features เหมือนกับ VS Code extension ทุกประการ รวมถึง Skills ครบทุกตัว
CLAUDE.md คืออะไร และต้องสร้างเองไหม?
CLAUDE.md คือไฟล์ที่วางไว้ใน root ของโปรเจค เพื่อบอก Claude ว่าโปรเจคนี้มี rules, conventions, หรือ context อะไรบ้างที่ต้องรู้

ถ้าไม่มี CLAUDE.md ก็ใช้งานได้ปกติ — Claude จะทำงานตาม default behavior แต่ถ้าสร้าง CLAUDE.md ขึ้นมา Claude จะ customize ตาม standard ที่ทีมกำหนดได้ เหมาะสำหรับ team projects ที่ต้องการ consistency

❌ Common Mistakes

❌ ใช้ Claude.ai web แล้วงงว่าทำไม Skills ไม่ทำงาน — หลายคนพิมพ์ /systematic-debugging ใน claude.ai web แล้วไม่เข้าใจว่าทำไม Claude ตอบแบบธรรมดาไม่ได้ invoke Skill
✅ Skills ทำงานได้เฉพาะใน Claude Code environments — CLI หรือ extension เท่านั้น ถ้าต้องการ Skills ให้ใช้ VS Code extension หรือ CLI
เหตุผล: claude.ai web เป็น product คนละตัว ไม่ได้ run Claude Code engine ที่ Superpowers Skills ถูก built บนนั้น
*(source: Claude Code documentation)*
❌ คิดว่า VS Code extension "น้อยกว่า" CLI และ feature บางอย่างถูกตัดออก
✅ VS Code extension รัน Claude Code engine เดียวกันกับ CLI — Skills ครบ 100% ทั้งสองให้ผลลัพธ์เหมือนกัน ข้อแตกต่างคือ UX ไม่ใช่ capability
เหตุผล: Extension เป็น UI wrapper บน Claude Code core ซึ่งเป็น engine เดียวกัน ไม่มี feature ไหนที่ถูกลดทอน
*(source: Anthropic Claude Code docs)*
❌ ไม่ได้สร้าง CLAUDE.md ในโปรเจค แล้วสงสัยว่าทำไม Claude ไม่รู้ conventions ของทีม
✅ สร้าง CLAUDE.md ไว้ใน project root เพื่อบอก Claude เรื่อง coding style, naming conventions, หรือ team standards ที่ต้องการ Claude จะอ่านไฟล์นี้ทุกครั้งโดยอัตโนมัติ
เหตุผล: Claude ไม่รู้ว่าทีมคุณใช้ conventions อะไร ถ้าไม่บอก มันจะ default ไปตาม general best practices ที่อาจไม่ตรงกับที่ทีมต้องการ
*(source: CLAUDE.md documentation)*
❌ Install VS Code extension แล้วรอให้ Claude เริ่มก่อนโดยไม่ได้บอก context — เปิด chat แล้วถามว่า "ช่วย fix bug หน่อย" โดยไม่ได้บอกว่า bug คืออะไร
✅ เลือก code ที่มีปัญหาก่อน แล้วค่อยถาม — หรือถ้าใช้ Skills ให้บอก context ใน prompt เช่น /systematic-debugging error: TypeError Cannot read property 'map' of undefined ที่บรรทัด 42
เหตุผล: แม้ Claude เห็น project context ทั้งหมด แต่การ provide context ที่ชัดเจนยิ่งทำให้ได้คำตอบที่แม่นยำขึ้นมาก
*(source: Claude Code best practices)*

🔧 Workshop — ลองทำตามได้เลย

🧪 Workshop 1 — Debug โค้ดด้วย /systematic-debugging ใน VS Code

  1. ติดตั้ง Claude Code extension จาก VS Code Marketplace (ดูขั้นตอนด้านบน)
  2. เปิดโปรเจคที่มี bug หรือสร้าง script ง่ายๆ ที่มีปัญหาไว้ทดสอบ
  3. เปิด Claude chat panel: Ctrl+Shift+P → "Claude Code"
  4. พิมพ์: /systematic-debugging โค้ดนี้ error: [อธิบาย error ที่เห็น]
  5. ตอบคำถาม Claude ทีละข้อ — Skill จะถามเพื่อ narrow down ปัญหา อย่า skip
  6. Apply fix ที่ Claude แนะนำโดยกด "Insert" หรือ copy เข้าไฟล์โดยตรง
Tips: ถ้า Claude ถามเรื่องที่คุณไม่รู้ให้บอกตรงๆ ว่า "ไม่รู้" — Skill ออกแบบมาให้ handle กรณีนี้ได้ จะ suggest วิธี investigate แทน

✅ Workshop 2 — Code Review ก่อน Commit ด้วย /code-review

  1. เขียน function หรือ feature ใหม่ให้เสร็จก่อน (ไม่ต้อง perfect)
  2. เปิดไฟล์ที่เขียนเสร็จใน VS Code
  3. เปิด Claude chat แล้วพิมพ์: /code-review ช่วย review ไฟล์นี้ก่อน commit
  4. Claude จะ review พร้อม severity: Critical / Warning / Info สำหรับแต่ละ issue
  5. แก้ไข Critical และ Warning ทุกข้อก่อน commit (Info ดูตามความเหมาะสม)
  6. ถ้าไม่เข้าใจ feedback ไหน ถามกลับใน chat ได้เลย: "ทำไม X ถึงเป็น Warning?"
Pro tip: สร้าง CLAUDE.md ในโปรเจคก่อน แล้วใส่ coding standards ของทีม จะทำให้ Claude review ตาม standard ที่คุณกำหนด ไม่ใช่แค่ general best practices

📌 สรุปบทที่ 6

ลองตอบคำถามด้านล่างด้วยตัวเองก่อนดูเฉลย — บทนี้เป็น "practical" ที่สุด เพราะมันคือสิ่งที่คุณใช้จริงในการทำงาน

คำถามที่ 1

อธิบายความแตกต่างหลักระหว่าง Claude.ai web กับ Claude Code สำหรับนักพัฒนา — ยกตัวอย่างงาน 2-3 อย่างที่แต่ละตัวเหมาะกว่ากัน

▶ ดูแนวคำตอบ
Claude.ai web เหมาะสำหรับ: ถาม-ตอบทั่วไป, เขียน essay หรือสรุปเอกสาร, brainstorm ไอเดีย — ทำงานจาก browser ไม่ต้อง install อะไร แต่ไม่มี Skills ไม่เห็น codebase และแก้ไขไฟล์โดยตรงไม่ได้

Claude Code (ผ่าน CLI หรือ VS Code extension) เหมาะสำหรับ: debug code, review pull request, refactor codebase, สร้างไฟล์ใหม่, invoke Skills เช่น /systematic-debugging — เป็น product คนละตัวที่ออกแบบสำหรับนักพัฒนาโดยเฉพาะ
คำถามที่ 2

ถ้าต้องการใช้ /systematic-debugging ใน VS Code ต้องทำขั้นตอนอะไรบ้างตั้งแต่ต้นจนถึงจุดที่ Skill เริ่มทำงาน?

▶ ดูแนวคำตอบ
1. ติดตั้ง Claude Code extension จาก VS Code Marketplace (ค้นหา "Claude Code")
2. Sign in ด้วย Anthropic account
3. เปิดโปรเจคที่มีปัญหาใน VS Code
4. เปิด Claude chat panel: Ctrl+Shift+P → "Claude Code" หรือกด icon ใน sidebar
5. พิมพ์ /systematic-debugging พร้อม context ของปัญหา → Skill เริ่มทำงานทันที
คำถามที่ 3

Skills ใน VS Code extension ทำงานต่างจาก CLI ไหม? เพราะอะไร? และถ้าทีมของคุณมีทั้ง CLI users และ VS Code users — พวกเขาจะได้ Skills คนละชุดกันไหม?

▶ ดูแนวคำตอบ
Skills ทำงานเหมือนกัน 100% เพราะ VS Code extension รัน Claude Code engine เดียวกับ CLI ที่ backend ข้อแตกต่างคือ UX เท่านั้น — CLI พิมพ์ใน terminal, VS Code พิมพ์ใน chat panel

ทีมที่มีทั้ง CLI users และ VS Code users จะได้ Skills ชุดเดียวกันทุกประการ สามารถทำงานร่วมกันได้ไม่มีปัญหา ตราบใดที่ใช้ Claude Code ทั้งคู่ (ไม่ว่าจะผ่าน interface ใดก็ตาม)
